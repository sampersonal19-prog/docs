The KenerateAI API provides programmatic access to powerful generative models such as GPT Image 2 Text-to-Image and GPT Image 2 Edit. Our API is organized around REST. It has predictable resource-oriented URLs, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

## Authentication

All API requests must be authenticated using a Bearer token. You can generate API keys from your [Developer Dashboard](https://kenerateai.com/developer/api-key).

To authenticate, provide your API key in the `Authorization` header of your HTTP request:

```bash
Authorization: Bearer YOUR_API_KEY
```

<Warning>
  Keep your API keys secure. Do not share them in publicly accessible areas such as GitHub, client-side code, etc.
</Warning>

## Content Type

Unless otherwise specified, all API requests should include the `Content-Type: application/json` header and all request bodies should be valid JSON.

## Base URL

The base URL for the KenerateAI REST API is:

```bash
https://api.kenerateai.com/api/v2
```

*(Note: We also support `/api/v3` as the latest endpoint revision with identical structures but newer backend optimizations)*
