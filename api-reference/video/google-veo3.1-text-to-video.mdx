Google Veo 3.1 Text-to-Video converts natural-language prompts into high-quality video clips with realistic motion, camera control, and optional synchronized audio. Use this endpoint for text-only video generation with landscape or portrait output.

## Endpoint

```http
POST https://api.kenerateai.com/api/v1/kenerateai/google-veo3.1-text-to-video
```

## Headers

<ParamField header="Authorization" type="string" required>
  Bearer authentication header. Example: `Bearer YOUR_API_KEY`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`.
</ParamField>

## Body

<ParamField body="prompt" type="string" required>
  Text prompt for generation. Describe the scene, subject, action, camera movement, lighting, and sound.
</ParamField>

<ParamField body="aspect_ratio" type="string" default="16:9">
  Aspect ratio of the generated video. Supported values: `16:9`, `9:16`.
</ParamField>

<ParamField body="duration" type="integer" default="8">
  Duration of the generated video in seconds. Supported values: `4`, `6`, `8`.
</ParamField>

<ParamField body="resolution" type="string" default="1080p">
  Video resolution. Supported values: `720p`, `1080p`, `4k`.
</ParamField>

<ParamField body="generate_audio" type="boolean" default="true">
  Whether to generate synchronized audio with the video.
</ParamField>

<ParamField body="negative_prompt" type="string">
  Elements to avoid in the generated video.
</ParamField>

<ParamField body="seed" type="integer">
  Random seed for reproducible generations. Supported range: `-1` to `2147483647`.
</ParamField>

<ParamField body="webhook_url" type="string">
  A URL that KenerateAI will send an HTTP POST request to when the video generation is complete.
</ParamField>

## Response

When the request is accepted, KenerateAI returns a queued prediction. The final video URL is available in `data.outputs` once the task is completed.

<ResponseField name="code" type="integer">
  HTTP status code.
</ResponseField>

<ResponseField name="message" type="string">
  Status message, such as `success`.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique identifier for the prediction.
    </ResponseField>
    <ResponseField name="model" type="string">
      Model ID used for this prediction.
    </ResponseField>
    <ResponseField name="outputs" type="array">
      Array of generated video URLs. Empty while the task is queued or processing.
    </ResponseField>
    <ResponseField name="urls" type="object">
      <Expandable title="properties">
        <ResponseField name="get" type="string">
          URL to poll for the prediction result.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="status" type="string">
      Current task status: `created`, `processing`, `completed`, or `failed`.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO timestamp for when the prediction was created.
    </ResponseField>
    <ResponseField name="error" type="string">
      Error message if the task failed.
    </ResponseField>
    <ResponseField name="timings" type="object">
      Timing information for the prediction.
    </ResponseField>
  </Expandable>
</ResponseField>

## Polling and Webhooks

Poll `data.urls.get` until `data.status` is `completed`, or pass `webhook_url` in the request body to receive a POST callback when generation finishes.

```bash
curl --request GET \
  --url https://api.kenerateai.com/api/v1/predictions/e42dc9f3-8b7c-4813-bfa5-985162a04944 \
  --header 'Authorization: Bearer YOUR_API_KEY'
```

## Pro Tips

1. Describe subject, motion, environment, camera movement, lighting, and audio cues in one focused prompt.
2. Use camera verbs such as `tracking`, `push in`, `pan`, or `handheld` for stronger motion direction.
3. Use `negative_prompt` for artifacts you want to avoid, such as blur, jitter, distortion, or low quality.
4. Choose `9:16` for mobile/social clips and `16:9` for cinematic or desktop playback.
5. Set `seed` when iterating so changes to other parameters are easier to compare.
